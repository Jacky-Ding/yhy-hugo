## Architecture Decisions

1. **i18n-first approach**: All translatable content lives in i18n TOML files. Page partials call `{{ T "key" }}` to render localized content. This centralizes all text for easy editing.

2. **Branch bundle routing**: Content sections (about, products, etc.) use Hugo's branch bundle system with `list.html` handling the routing to appropriate partials based on `.Section`.

3. **Language detection**: `lang-detect.js` uses IP geolocation API (ipregistry) for accurate region-based language switching, with browser Accept-Language as fallback. Result cached in localStorage.

4. **Responsive CSS**: Single `style.css` with 3 breakpoints (desktop, tablet <900px, mobile <600px) using CSS Grid and flexbox.
